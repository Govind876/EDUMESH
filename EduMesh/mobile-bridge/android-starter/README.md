# SchoolSync Android Bridge Starter

This starter shows how an Android client can consume the VCD QR payload generated by the Go backend and download the SchoolSync artifact offline.

## What this starter includes

- `VcdPayload.kt`: Parses and validates QR JSON payload.
- `WifiDirectVcdManager.kt`: Wi-Fi Direct group discovery/connect skeleton using `WifiP2pManager`.
- `VcdDownloader.kt`: Downloads the binary/apk over local HTTP and saves to app external files.
- `AndroidManifest.permissions.snippet.xml`: Required permissions snippet.

## Backend contract used

The starter expects the QR payload shape currently returned by backend field `qrPayload` from `POST /vcd/start`:

```json
{
  "type": "schoolsync-vcd",
  "downloadUrl": "http://192.168.49.1:8080/download?token=...",
  "serverIp": "192.168.49.1",
  "port": 8080,
  "ssid": "SchoolSync-Direct",
  "password": "12345678",
  "bluetoothFallback": {
    "enabled": true,
    "mode": "OPP"
  }
}
```

Use `artifactPath` in `/vcd/start` when sender should distribute an APK instead of the running executable:

```json
{
  "ssid": "SchoolSync-Direct",
  "password": "12345678",
  "serverIp": "192.168.49.1",
  "artifactPath": "E:\\builds\\schoolsync.apk",
  "enableBluetoothFallback": true
}
```

## Integration steps (Android app)

1. Copy Kotlin files into your Android module, e.g. `app/src/main/java/.../vcd/`.
2. Add the permissions from `AndroidManifest.permissions.snippet.xml` to `AndroidManifest.xml`.
3. Build a QR scan screen:
   - Scan string.
   - Parse with `VcdPayloadParser.parse(raw)`.
   - Start Wi-Fi Direct flow with `WifiDirectVcdManager`.
4. On successful peer/group connection, call:
   - `VcdDownloader.download(payload.downloadUrl, "schoolsync.apk")`
5. Trigger install intent for APK:
   - `Intent.ACTION_VIEW`
   - `content://` URI from `FileProvider`
   - MIME `application/vnd.android.package-archive`
6. If Wi-Fi Direct fails and payload says bluetooth fallback is enabled:
   - Switch to Bluetooth OPP sender/receiver flow.

## Notes

- Android cannot auto-join Wi-Fi Direct from plain QR alone in all OS versions; user confirmation is often required.
- For Android 13+, nearby and Wi-Fi permissions are runtime-sensitive; request them before discovery.
- For production, add:
  - Download integrity check (`SHA-256` metadata and verification).
  - TLS for non-lab deployments.
  - Expiry UX (backend session currently expires after 10 minutes).
